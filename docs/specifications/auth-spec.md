# COMPACT STOCKER 認証仕様

## 1. 認証の目的

- 利用者を識別する
- 所属倉庫を確定する
- 参照可能なデータ範囲を倉庫単位で制御する

## 2. 現在の実装

- ユーザー ID とパスワードの入力を受け付ける
- 両方が非空であればログイン成功とする
- 実運用向け認証は未実装

## 3. 将来の認証フロー

1. ユーザーが ID とパスワードを入力する
2. アプリが Azure 上の Web API に認証要求を送る
3. API がユーザー情報を返す
4. レスポンスに含まれる所属倉庫情報を確定する
5. 以降の画面は所属倉庫のデータを参照する

## 4. 認証後に保持したい情報

- accessToken
- userId
- userCode
- userName
- warehouseId
- warehouseCode
- warehouseName

## 5. 画面側の影響

- ログイン成功時点で倉庫が確定する
- 倉庫選択 UI は基本不要とする
- 画面側はログイン済み倉庫のデータのみを前提に表示する

## 6. 例外ケース

- 認証失敗
  - エラーメッセージを表示してログイン画面に留まる
- 所属倉庫未設定
  - 利用不可メッセージを表示する
- トークン期限切れ
  - 再ログインを促す

## 7. 実装方針メモ

- 現在の `TokenManager` は将来のトークン保持先として流用可能
- 本実装時は、ユーザー情報と倉庫情報の保持方法を見直す
