# COMPACT STOCKER MVP定義

## 1. 目的

この文書は、最初に公開可能な品質まで持っていくための最小機能範囲を定義するものです。

## 2. MVPの考え方

MVP では、在庫確認から出荷作業、出荷実績確認までの基本導線が迷わず使えることを優先します。

公開時点では、以下を重視します。

- 主要画面が破綻なく使える
- 画面遷移が自然である
- 一連の業務フローが理解できる
- 未完成機能に見える表現を減らす

## 3. MVPに含める機能

- ログイン
- ホーム画面
- 在庫一覧
- 在庫履歴確認
- 出荷作業開始
- 出荷プラン一覧
- 出荷品目検索
- 出荷実績入力
- 出荷実績検索
- 出荷実績一覧
- CSV 出力
- QR コード読み取り

## 4. MVPで許容する制約

- サンプルデータの利用
- 簡易ログイン
- 実績データのローカル管理
- 倉庫固定の疑似データ表示

## 5. MVPで含めない機能

- 本番認証
- Azure API 接続
- 倉庫切替 UI
- リアルタイム同期
- 権限管理
- 通知機能
- 管理者用画面
- マスタメンテナンス

## 6. 公開前に必要な状態

- iPhone 実機で主要画面が問題なく動く
- App Store 説明文と実装内容が一致している
- テスト用、デモ用、試作品と受け取られにくい状態になっている
- ログインから主要画面までの導線が詰まらない
- カメラ権限が自然に説明される

## 7. MVP後に進めるもの

- Azure 連携
- 所属倉庫に基づくデータ切替
- 永続化の本格導入
- 認証方式の正式化
- iPad 最適化
- 無料公開版向け Azure デモ制限
- 組織向け有料版の運用条件整理
