# COMPACT STOCKER 非機能仕様

## 1. 品質方針

- 少ない操作で主要機能へ到達できること
- 日本語 UI で直感的に理解できること
- 実機で違和感なく操作できること

## 2. UX 方針

- 出荷作業は作業者が迷わない導線にする
- 一覧は情報量と読みやすさのバランスを取る
- 操作結果は画面上で分かるようにする

## 3. パフォーマンス方針

- 初期表示で待たされる感覚を減らす
- 一覧画面はスクロールが滑らかであること
- QR 読み取り導線はすぐ使えること

## 4. 対応方針

- 主対象は iPhone
- 現在の iOS アプリは iPhone 専用設定とする
- iPad は現時点ではサポート対象外だが、将来対応候補として扱う

## 5. セキュリティ方針

- カメラは QR 読み取り用途のみに利用する
- 認証導入後はログイン情報を安全に扱う
- 必要以上の個人情報は持たない
- ローカル LAN 接続を許可する場合でも HTTPS を必須とする
- ATS の包括例外に依存しない
- ローカルネットワーク利用時は、用途が分かる権限説明文を用意する

## 6. 公開方針

- App Store 文言は実装済み機能だけを記載する
- ベータ版、テスト版、試作品と受け取られる表現を避ける
- 公開前に主要導線を実機で確認する
- LAN 接続機能を含む場合は、審査時に接続条件と確認方法を説明できる状態にする

## 7. 保守方針

- 仕様変更時は仕様書を更新する
- App Store 審査で指摘を受けたら、文面と実装の両面を見直す
- サンプルデータと将来 API 連携の責務を分けて管理する
- Docker テスト用設定と Azure 本番設定を分離して管理する
