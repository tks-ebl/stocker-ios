# COMPACT STOCKER 非機能仕様

## 1. 品質方針

- 少ない操作で主要機能へ到達できること
- 日本語 UI で直感的に理解できること
- 実機で違和感なく操作できること

## 2. UX 方針

- 出荷作業は作業者が迷わない導線にする
- 一覧は情報量と読みやすさのバランスを取る
- 操作結果は画面上で分かるようにする

## 3. パフォーマンス方針

- 初期表示で待たされる感覚を減らす
- 一覧画面はスクロールが滑らかであること
- QR 読み取り導線はすぐ使えること

## 4. 対応方針

- 主対象は iPhone
- iPad は現時点では優先度を下げる
- App Store 審査時に iPad 対応が必要なら、表示崩れを防ぐ対応を行う

## 5. セキュリティ方針

- カメラは QR 読み取り用途のみに利用する
- 認証導入後はログイン情報を安全に扱う
- 必要以上の個人情報は持たない

## 6. 公開方針

- App Store 文言は実装済み機能だけを記載する
- ベータ版、テスト版、試作品と受け取られる表現を避ける
- 公開前に主要導線を実機で確認する

## 7. 保守方針

- 仕様変更時は仕様書を更新する
- App Store 審査で指摘を受けたら、文面と実装の両面を見直す
- サンプルデータと将来 API 連携の責務を分けて管理する
