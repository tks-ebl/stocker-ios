# COMPACT STOCKER Azure デモ制限 API 仕様

## 1. 文書の目的

この文書は、無料公開版が接続する Azure デモ環境に対して適用する利用制限と、Web API 側のレスポンス仕様を定義するものです。

iOS 側と Web API 側で同じ制限前提を共有し、利用者への案内文言を揃えることを目的とします。

## 1.1 現在の実装状況

- この文書の制限仕様は構想段階
- 現在の `StockerWebAPI` では未実装
- 実装状況は `implementation-gap-notes.md` に追記する
- 制御は初期版から入れる方針を採用する

## 2. 制限対象

- Azure デモ環境のみ
- 社内サーバー運用時には適用しない

## 3. 固定制限値

- 利用可能倉庫数
  - 1 倉庫のみ
- 利用可能ユーザー数
  - 2 ユーザーまで
- 在庫データ件数
  - 100 件まで
- 履歴参照期間
  - 直近 30 日まで
- CSV 出力
  - 利用不可

## 4. 制限の適用方針

- 制限は Web API 側で判定する
- iOS 側では制限値を案内表示する
- 制限超過時は HTTP ステータスとエラーコードを返す
- エラーレスポンスには、利用者向けメッセージと開発者向け識別子を含める

## 5. API 別制限

### 5.1 ログイン

- Azure デモ環境では 2 ユーザーのみ利用可能とする
- 許可されたデモユーザー以外は認証成功させない

### 5.2 在庫一覧

- 返却件数は最大 100 件とする
- 101 件目以降は返却しない
- 総件数情報を返す場合は、上限適用済みであることが分かる補足を付与する

### 5.3 在庫履歴

- 直近 30 日より古い履歴は返却しない
- 期間外データ要求時は空配列返却ではなく、制限説明用メタデータを返せるようにする

### 5.4 出荷実績 CSV

- Azure デモ環境では CSV 出力 API を利用不可とする
- CSV 出力要求時は制限エラーを返す

### 5.5 倉庫関連

- 倉庫は 1 倉庫固定とし、他倉庫 ID へのアクセスは不可とする
- 不一致な `warehouseId` には認可エラーまたは制限エラーを返す

## 6. エラーレスポンス共通仕様

### 6.1 HTTP ステータス

- 認証失敗
  - `401 Unauthorized`
- 許可対象外倉庫や契約外アクセス
  - `403 Forbidden`
- デモ制限超過
  - `403 Forbidden`
- 入力不正
  - `400 Bad Request`

### 6.2 JSON 形式

```json
{
  "errorCode": "DEMO_LIMIT_CSV_DISABLED",
  "message": "Azureデモ環境ではCSV出力を利用できません。",
  "details": {
    "limitType": "csv",
    "environment": "azure-demo"
  }
}
```

### 6.3 エラーコード一覧

- `DEMO_LIMIT_USER_NOT_ALLOWED`
  - デモ許可外ユーザー
- `DEMO_LIMIT_WAREHOUSE_NOT_ALLOWED`
  - デモ許可外倉庫
- `DEMO_LIMIT_INVENTORY_CAPPED`
  - 在庫件数上限到達
- `DEMO_LIMIT_HISTORY_RANGE`
  - 履歴参照期間外
- `DEMO_LIMIT_CSV_DISABLED`
  - CSV 出力禁止

## 7. iOS 側表示方針

- `message` をそのまま一次表示に利用できるようにする
- 必要に応じて画面側で補足文言を追加する
- 制限エラーは通信障害と分けて扱う

### 7.1 表示文言案

- `Azure デモ環境ではこの操作は利用できません`
- `Azure デモ環境では 2 ユーザーまで利用できます`
- `Azure デモ環境では在庫は 100 件まで表示されます`
- `Azure デモ環境では履歴は直近 30 日まで表示されます`
- `CSV 出力は社内サーバー環境で利用できます`

## 8. レスポンス拡張案

- `GET /health` に環境識別を含める
- `GET /auth/login` または初期設定 API で制限情報を返す

例:

```json
{
  "environment": "azure-demo",
  "limits": {
    "warehouseCount": 1,
    "userCount": 2,
    "inventoryCount": 100,
    "historyDays": 30,
    "csvExport": false
  }
}
```

## 9. 実装メモ

- 制限値は設定ファイルまたは環境変数で管理できるようにする
- ただし無料公開版の正式値は本仕様の固定値を既定とする
- デモ制限の分岐は iOS 側ではなく API 側で一元管理する
